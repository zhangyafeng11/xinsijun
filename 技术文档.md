# 新四军抗战文化导学系统 - 技术文档

## 系统概述

本系统是一个基于 localStorage 的学习记录系统，实现了"公众号学习 + 网页成长记录"的导学闭环。

### 技术栈
- **前端**：HTML5 + CSS3 + 原生 JavaScript
- **数据存储**：localStorage（无需后端、无需数据库）
- **设计风格**：红色文化数字展馆风格

---

## 一、localStorage 数据结构

### 存储键名
```javascript
const STORAGE_KEY = 'newfourtharmy_learning';
```

### 数据结构
```javascript
{
  "timelineCompleted": false,      // 时间轴学习是否完成
  "heroesCompleted": false,       // 人物学习是否完成
  "videoCompleted": false,         // 微课学习是否完成
  "reflectionCompleted": false,    // 学习反思是否完成
  "activeKeywords": [],           // 已激活的精神关键词数组
  "completedTasks": []            // 已完成的任务ID数组
}
```

### 字段说明

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `timelineCompleted` | Boolean | 时间轴学习（军部组建）是否完成 | `false` |
| `heroesCompleted` | Boolean | 人物学习（叶挺、粟裕）是否完成 | `false` |
| `videoCompleted` | Boolean | 微课学习（黄桥战役）是否完成 | `false` |
| `reflectionCompleted` | Boolean | 学习反思是否提交 | `false` |
| `activeKeywords` | Array | 已激活的精神关键词ID数组 | `[]` |
| `completedTasks` | Array | 已完成的任务ID数组 | `[]` |

```javascript
function updateCircleProgress(circleId, valueId, percentage) {
  const circle = document.getElementById(circleId);
  const value = document.getElementById(valueId);
  
  circle.style.background = `conic-gradient(#c02425 ${percentage * 3.6}deg, #f5c518 ${percentage * 3.6}deg 360deg)`;
  value.textContent = percentage + '%';
}
```

## 六、学习报告统计规则

### 统计指标

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 已完成学习数 | (timelineCompleted ? 1 : 0) + (heroesCompleted ? 1 : 0) + (videoCompleted ? 1 : 0) | 0-3 |
| 已获得徽章 | (completedCount > 0 ? 1 : 0) + (videoCompleted ? 1 : 0) + (reflectionCompleted ? 1 : 0) | 0-3 |
| 已激活关键词 | activeKeywords.length | 0-8 |
| 当前学习阶段 | 根据已完成学习数和反思完成情况判断 | 入门/初级/进阶/精通 |

### 学习阶段判断规则

```javascript
let stage = '入门';
if (completedCount >= 3 && data.reflectionCompleted) {
  stage = '精通';
} else if (completedCount >= 2) {
  stage = '进阶';
} else if (completedCount >= 1) {
  stage = '初级';
}
```

## 七、URL参数处理流程

### 1. 参数读取
```javascript
const params = new URLSearchParams(window.location.search);
const task = params.get('task');
```

### 2. 任务验证
检查 `task` 是否在 `TASK_MAPPING` 中存在

### 3. 数据更新
- 标记对应任务完成
- 激活相关关键词
- 记录到已完成任务列表

### 4. UI更新
- 显示成功弹窗（3秒后自动消失）
- 更新进度条和环形进度
- 激活关键词
- 点亮徽章
- 更新学习报告

### 5. URL清理
```javascript
window.history.replaceState({}, document.title, window.location.pathname);
```

## 八、弹窗系统

### 弹窗类型

#### 1. 学习成功弹窗（successModal）
- **触发时机**：通过URL参数完成学习任务
- **显示内容**：任务名称
- **自动关闭**：3秒后

#### 2. 徽章获得弹窗（badgeModal）
- **触发时机**：提交学习反思后
- **显示内容**：徽章图标和名称
- **手动关闭**：点击确定按钮

#### 3. 学习报告弹窗（reportModal）
- **触发时机**：点击"生成学习报告"按钮
- **显示内容**：成功提示
- **手动关闭**：点击确定按钮

#### 4. 重置确认弹窗（resetModal）
- **触发时机**：点击"重新学习"按钮
- **显示内容**：确认提示
- **手动关闭**：点击确定或取消按钮

## 九、重置功能

### 重置流程
1. 用户点击"重新学习"按钮
2. 显示确认弹窗
3. 用户确认后清空 localStorage
4. 刷新页面
5. 显示"学习记录已重置"提示

### 重置代码
```javascript
function confirmReset() {
  localStorage.removeItem(STORAGE_KEY);
  closeModal('resetModal');
  
  setTimeout(() => {
    location.reload();
  }, 500);
}
```

## 十五、常见问题

### Q1: 学习记录会丢失吗？
A: 学习记录存储在浏览器的 localStorage 中，清除浏览器数据会丢失记录。

### Q2: 可以在多个设备上同步吗？
A: 不可以，localStorage 是本地存储，无法跨设备同步。如需跨设备，需要后端支持。

### Q3: URL参数会被搜索引擎收录吗？
A: 系统会在记录完成后自动清除URL参数，避免搜索引擎收录。

### Q4: 如何修改学习进度百分比？
A: 在 `updateUI()` 函数中修改 `timelineProgress`、`heroesProgress`、`videoProgress` 的值。

### Q5: 如何添加新的学习任务？
A: 在 `TASK_MAPPING` 中添加新任务，并在 `handleURLParams()` 中添加对应的处理逻辑。
